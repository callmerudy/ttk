Toolkit Demo Bugs

  1. Conversion error on TimeBank-1-pre/ABC19980108.1830.0711
    2. Reported: 10/29/07
    Status: fixed for this document

    When processing TimeBank-1-pre/ABC19980108.1830.0711, with document type timebank-preprocessed, the code that converts into the shallow tree throws an error. This error does not occur with TimeBank/ABC19980108.1830.0711.xml, it also does not occur with TimeBank-1-pre/ABC19980114.1830.0611

    10/30/07:
    • This was caused by an error in Evita, which takes a token "owns" and marks up the 's' as an event. There may to be some interplay with the VX tags or some of the other tags that go beyond the minimum set of NG and VG.
    • The document was replaced by one created with the toolkit preprocessor. But other documents with extra tags remain in the directory. Many of them seem to work, but the error was not truly fixed so other documents could croak.
  2. Preprocessor error on TimeBank/AP900815-0044.xml
    3. Reported: 10/30/07
    Status: fixed for this document

    Seems like an exception in the unpacking code of the converter. This error does not occur on the other AP document.

    10/30/07:
    • This was caused by stray XML tags in the document. The tokenizer leaves them glued on, the tagger creates items like "<IN-MW>according/VBG to/TO </IN-MW>diplomatic/JJ" and the chunker introduces crossing tags: "<VG><IN-MW>according/VBG</VG> to/TO </IN-MW>diplomatic/JJ". The formatConversor chokes on these.
    • This was fixed for this document by removing the offending tags when first creating the fragment.
    • Needs a more robust solution
  3. No event in "He'll sleep"
    4. Reported: 10/30/07
    Status: not fixed

    This is a problem with the tagger, which tags 'll' and 'sleep' as nouns. All is fine with "He will sleep". What also works is "He'll vindicate us".
  4. ATEE document break in an early processing phase
